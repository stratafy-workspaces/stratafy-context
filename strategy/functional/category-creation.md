# Category Creation

## Description
Category Creation defines 'Strategic Context Layer' as the new infrastructure category for the AI era — the structured, queryable, always-current representation of organisational intent that makes strategic context available wherever decisions are being made, by humans or AI. The framing positions Stratafy as infrastructure (context layer that shows up where work happens) rather than application (tool you visit). The 'Strategic Operating System' label remains as the broader category name for market-facing communications, while 'Strategic Context Layer' is the technical/investor thesis. Core narrative: capability is commoditised, tasks are well-defined, but context is scarce — the organisation that solves context wins. Existing strategy tools are Mode 1 only (better filing cabinets); Stratafy provides the context layer that every person, tool, and agent draws on. The competitive moat is the consumption-as-maintenance flywheel: every use of strategy either confirms accuracy or reveals gaps, creating compounding intelligence that increases switching costs.

## Strategic Intent

**Question answered**: Can we define "Strategic Operating System" as a recognized category?

**Time Horizon**: Q2-Q4 2026

## Content Sequence: Problem Education First

We don't claim the category before earning the right. The content sequence is deliberate:

| Phase | Content Focus | Purpose |
|-------|---------------|---------|
| **1. Problem** | "The Strategy Execution Gap" | Establish the problem is structural |
| **2. Reframe** | "Identity Is Infrastructure" | Shift from culture to infrastructure |
| **3. Shift** | "Quarterly to Continuous" | Establish the cadence change |
| **4. Solution** | "Strategic Operating System" intro | Only NOW introduce the category |
| **5. Proof** | Case studies, dogfooding examples | Show, don't tell |

**The principle**: Problem education earns the right to define the category. We don't promote solutions until we've demonstrated deep understanding of the problem.

This is "show, don't tell" applied to category creation itself.

## Positioning

- Own the term "Strategic Operating System"
- Frame the problem as "execution gap + AI alignment gap"
- Position against: static strategy documents, OKR tools, project management, consulting
- Lead with problem education, not solution promotion (per our values)

## Messaging Principle

**Lead with outcomes, not infrastructure.**

| Don't Say | Do Say |
|-----------|--------|
| "Strategic operating system with 4 layers" | "2-3 days of strategic work in 2 hours" |
| "Foundation + Strategy + MCP architecture" | "AI that knows your full context and writes directly into your system" |
| "Alignment scoring algorithm" | "Know if your strategy and values are actually connected" |
| "Full automation of strategy" | "Human judgment amplified by AI, never replaced" |

Infrastructure is the "how" — reveal it when they ask. Outcomes are the "why" — lead with them always.
