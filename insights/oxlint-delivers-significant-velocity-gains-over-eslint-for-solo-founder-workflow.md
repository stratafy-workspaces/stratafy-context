# oxlint delivers significant velocity gains over eslint for solo founder workflow

## Summary
oxlint provides dramatically faster linting than eslint, compressing the dev feedback loop. For a solo founder maximising iterations per day, this tooling choice compounds — faster lint cycles mean more shipping velocity with the same effort.

## Description
Switching from eslint to oxlint revealed a significant performance difference in the linting step of the development workflow. oxlint, built in Rust, runs orders of magnitude faster than eslint for equivalent rule sets. For a solo founder where every minute of dev cycle time matters, this kind of tooling velocity gain compounds across hundreds of daily iterations. This reinforces the broader pattern: evaluate Rust-based tooling replacements across the stack wherever JavaScript tooling creates friction. The insight connects to the AI Leverage and Automation strategy — choosing tools that maximise throughput per unit of founder time.
