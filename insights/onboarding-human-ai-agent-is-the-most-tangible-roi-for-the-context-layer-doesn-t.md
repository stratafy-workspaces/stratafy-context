# Onboarding (human + AI agent) is the most tangible ROI for the context layer — doesn't require full thesis buy-in

## Summary
Context layer document v2 identifies onboarding as a standalone value proposition: human time-to-productivity from 6-8 weeks to days, AI agent onboarding from manual prompt curation to context profile assignment. Measurable, doesn't require full category-creation buy-in. Strong proof point for STIR and investor metrics.

## Description
The context layer document describes onboarding (both human and AI agent) as one of the most tangible, measurable value propositions. Time-to-productivity for new hires goes from 6-8 weeks (osmosis, conversations, outdated wikis) to days (role-appropriate context package on day one). AI agent onboarding goes from manual system prompt curation to automated context profile assignment.

This matters because:
1. It doesn't require buying the full category-creation thesis — it works as standalone ROI
2. It's measurable (time-to-productivity is something every scaling company tracks)
3. It applies to both humans and AI agents (the same mechanism serves both)
4. It's immediately relevant to STIR and any growing company

For the STIR building blocks sequence, onboarding is a natural Block 2.5 or Block 3 proof point: when STIR brings on a new team member or contractor, can the context layer compress their ramp-up time? If yes, that's a concrete, measurable win that doesn't require understanding the full architecture.

For investor conversations: "Our customers reduce new hire time-to-productivity by 60%" is a metric that speaks for itself, regardless of whether the listener understands the context layer thesis.
