# Customer onboarding must follow a progressive building blocks sequence, not expose full vision

## Summary
STIR engagement reveals the right approach: Foundation → Founder co-working → Team consumption → Living system. Each block solves an immediate problem, earns trust for the next, and validates a specific product thesis. Full context layer vision is the destination, not the pitch.

## Description
The strategic context layer document (v2) describes the complete vision — multi-resolution context generation, external consumption, autonomous agents, governance frameworks. But STIR doesn't need the full vision. They need building blocks delivered in sequence:

Block 1 (Weeks 1-2): Foundation + Strategy Architecture — feels like a facilitated strategy session. Familiar. Non-threatening. Immediate value = clarity and forced alignment conversations.

Block 2 (Weeks 2-4): Founder co-working experience — Mode 2 consumption. Founder prepares for real work (pitch, hiring, partnership) with strategic context present. This is where the "aha" happens — not because you explained the context layer, but because they felt it.

Block 3 (Weeks 4-6): Team consumption — expand beyond founder. Designer sees brand values, account manager sees client strategy, ops sees initiative priorities. First real test of context layer thesis at customer scale.

Block 4 (Weeks 6-8): Living system — flywheel starts turning. Gaps surface through use. Architecture updates. Maintenance through consumption, not scheduled reviews. This is the week-8 measurement: is the flywheel turning?

Each block does triple duty: value for customer (progressive, never overwhelming), validation for product (workbench → Mode 2 → role-based consumption → flywheel), evidence for investors (demonstrated sequence, not theoretical claims).

Critical discipline: don't show the full vision document to customers. Show them Block 1. Earn Block 2. Let the vision reveal itself through experience.
