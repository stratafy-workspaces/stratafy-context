# Strategic Context Layer narrative is the evolved category framing

## Summary
The Strategic Context Layer document articulates a sharper, more defensible framing than "Strategic Operating System" — positioning Stratafy as infrastructure (context layer) rather than application (operating system). The two-layer model (Workbench + Strategic Layer), three consumption modes, four resolution levels, and consumption-as-maintenance flywheel create a complete product thesis.

## Description
Key narrative evolutions from the document:

1. **Core framing shift**: From "Strategic Operating System" to "Strategic Context Layer" — infrastructure that shows up where work happens, not a destination you visit.

2. **Two-layer architecture**: Strategy Workbench (UI for building/maintaining) + Strategic Layer (MCP/API/conversational consumption everywhere). Clean separation of concerns.

3. **Three modes**: Mode 1 (Workbench), Mode 2 (Conversational/Integrated), Mode 3 (Autonomous Agents). Each serves different relationship between human and strategy.

4. **Four resolution levels**: Full, Strategic, Operational, Minimal — matching context depth to consumer need.

5. **External consumption as product capability**: Permissioned context packages for lawyers, advisors, investors, partners — not just GTM but a product feature with access scopes and governance.

6. **Consumption-as-maintenance flywheel**: The core defensibility thesis — every use of strategy either confirms accuracy or reveals gaps. Usage IS maintenance. This is the mechanism for operational lock-in.

7. **Competitive positioning**: Existing tools are Mode 1 only (better filing cabinets). The context layer is infrastructure — roads, not buildings.

Implications for strategy tree: Category Creation framing needs updating, Product Architecture needs the two-layer model, external context packaging needs a product initiative, consumption-as-maintenance needs elevation to principle/belief.
