# MCP permission gateway is the third infrastructure proof point — shows Stratafy as platform others build on

## Summary
Evolve permission gateway proves multi-tenancy, infrastructure-grade API surface, and distribution multiplier. Third proof point alongside workspace and Chat SDK for investor infrastructure thesis.

## Description
The Evolve MCP permission gateway isn't just a customer feature — it's a third infrastructure proof point for the investor demo alongside the workspace and Chat SDK.

What it proves:

1. Multi-tenancy is real: One Stratafy deployment serves multiple organizations with proper access boundaries. Strategy architecture isn't just internal — it's a shared context layer between advisory firms and clients, VCs and portfolio companies, consultancies and engagements.

2. API surface is infrastructure-grade: Evolve programmatically grants, scopes, and revokes access to strategic intelligence. That's a platform primitive (like Stripe's payment infrastructure or Twilio's communication infrastructure), not a SaaS feature.

3. Distribution multiplier is tangible: One Evolve relationship provides access to ~415 family offices. Every advisory firm, VC, and consultancy is a distribution node bringing 10-50x client deployments. The permission gateway makes that safe and scalable.

For the investor demo, the three-layer reveal becomes: "Here's the workspace (Act 1). Here's the same context in Slack and Google Chat (Act 2a — Chat SDK). And here's an advisory firm granting their clients scoped access through our permission gateway (Act 2b — platform primitive)." Each layer expands the aperture further.

This maps to the same pattern as Stripe (payment infrastructure others build on), Twilio (communication infrastructure others build on), Segment (data infrastructure others build on). Stratafy becomes strategy infrastructure others build their client relationships on top of.
