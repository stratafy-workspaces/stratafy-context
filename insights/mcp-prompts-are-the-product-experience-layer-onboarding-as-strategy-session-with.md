# MCP prompts are the product experience layer — onboarding-as-strategy-session with progressive disclosure and dual-path entry

## Summary
MCP prompts transform from developer tools into guided product workflows. Onboarding IS the strategy session — pre-populate from public sources, validate conversationally, customer leaves with real strategy. Two paths (methodology import + framework-free), progressive disclosure prevents adoption stall, every role becomes a sensing surface.

## Description
SYNTHESIZED FROM 6 INSIGHTS (Feb 8-9, 2026): 72abf1ed, 54a80ee1, 64e4b96b, dd783ef2, 663ab6f7, da60f44e

CORE ARCHITECTURE: MCP prompts are not just onboarding — they're a guided workflow layer that transforms MCP from developer tool into product experience. Each slash command combines context + instructions + links into a purposeful conversation. The demo IS the onboarding IS the strategy session. This is a structural advantage nobody else can replicate.

ONBOARDING-AS-STRATEGY-SESSION: Pre-populate workspace from public sources (website, LinkedIn), run a validation conversation, and the customer leaves with their real strategy in the system. The Viridis demo proved this: website was misaligned with actual strategy, which itself was a powerful finding. Every demo creates value regardless of outcome.

TWO PATHS: (1) Methodology import — upload OPSP/V/TO/OKR spreadsheet to auto-populate. Data entry burden drops to near zero. (2) Framework-free — build from first principles using Stratafy's native architecture. Path 2 users are stronger long-term (no methodology dependency, native Stratafy users from day one).

PROGRESSIVE DISCLOSURE: Don't surface all layers on day one. Start with foundation and strategy tree, unlock risks when data supports it, then assumptions, then radar. Complexity emerges naturally. Prevents Phase 2 adoption stall where users treat Stratafy as "fancy tracker."

SENSING NETWORK: Role-specific onboarding captures different intelligence from each role — founders reveal intent, managers reveal friction, ICs reveal ground truth. Every onboarding conversation is a data point feeding the feedback loop.

PUBLIC NARRATIVE SCAN: Feed in website, LinkedIn, job postings — compare against actual strategy. The gap between public narrative and internal reality is itself a strategic risk. Works as demo opener, standalone scan type, paid deliverable, and ongoing monitoring capability.
