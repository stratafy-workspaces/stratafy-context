# Stratafy is three inseparable layers unified by a first-principles opinion about strategy

## Summary
Stratafy is not a platform, not infrastructure, not a SaaS tool. It is three inseparable layers: (1) first-principles infrastructure — a typed, opinionated schema for how strategy actually works; (2) a platform of tools built on top; and (3) a continuously operating layer (cron jobs, health checks, distillation, role-based context packaging) that keeps the system alive. The layers are only valuable together, unified by an opinion that no generic infrastructure player can replicate.

## Description
Stratafy is defined by three inseparable layers, unified by a single first-principles opinion about what strategy is and how it works.

**Layer 1 — Infrastructure (first principles)**
The schema itself. Typed entities with domain semantics that no generic memory system has. A risk carries likelihood, impact, a score, a mitigation status. An assumption has confidence and impact-if-wrong. A decision is Type 1 or Type 2. The Foundation → Strategy → Initiatives → Objectives hierarchy is not configurable — it is the architecture of strategy, encoded as the data model. The intelligence layers (Risks, Assumptions, Decisions, Insights, Signals) and their relationships are an opinion about how strategy actually behaves. This is the foundation everything else sits on.

**Layer 2 — Platform**
The tools, interfaces, and workflows built on top of the infrastructure. Chat, Alignment Scanning, Radar, Cortex, Story Mode, InsightSync, Pulse, Context Matrix, Graph, Finance, Organogram, MCP. These are the surfaces through which humans and AI agents interact with and act on the infrastructure. They are opinionated too — but they are expressions of the infrastructure, not the infrastructure itself.

**Layer 3 — Operating System**
The continuous processes that keep everything alive without being asked. Distillation cron (every 5 minutes, change-aware), health scoring, content quality flags, role-based context packaging, permission governance, staleness decay, delta context. This is what separates Stratafy from a well-designed database with a nice UI. The system is alive — it runs, updates, detects, and surfaces continuously.

**Why the combination is defensible**
The three layers are only valuable together. Infrastructure without the operating layer goes stale. A platform without the infrastructure competes on features. An operating layer without domain infrastructure has nothing meaningful to operate on. Generic infrastructure plays (HydraDB, Mem0, Zep) have Layer 3 primitives and no Layer 1 opinion. They cannot replicate Stratafy from the infrastructure side up — the opinion must be built in from the start, not added later.

**The definition this produces:**
Stratafy is an opinionated strategic intelligence system — first-principles infrastructure for how strategy works, a platform of tools built on top, and a continuously operating layer that keeps strategic context alive, governed, and consumable by both humans and AI.
