# Claude-native delivery excites more than a standalone website

## Summary
EO Forum audience responded with unexpected excitement to Stratafy running inside Claude Desktop rather than as a website. The pull was not the platform UI — it was AI with strategic context, working inside a tool they're already exploring. This suggests the delivery model (Claude-native plugins) is part of the value proposition, not just a technical detail.

## Description
During the EO Forum Deep Dive presentation (March 2026), the audience showed more excitement about Stratafy operating inside Claude than about the website or platform itself. The website was never demoed — Leonard spoke about it and showed use cases directly in Claude Desktop. Despite never seeing the web interface, the room was engaged and excited.

This signals that the Claude-native approach is a differentiator, not a limitation. The audience sees value in AI that has their strategic context and works inside a tool they're already starting to adopt. A standalone SaaS website feels like "another tool to log into." Claude with Stratafy context feels like "my strategist, my CFO, my radar — inside the tool I already use."

Implications for positioning:
- Lead with "AI that knows your strategy" rather than "strategy platform"
- The delivery model (plugins in Claude) should be front and centre in messaging, not hidden as a technical implementation detail
- This may reduce friction to adoption — no new login, no new interface to learn
