# Semantic context assembly is a compounding moat — advantage widens as workspaces mature, invisible to competitors

## Summary
Naive context injection degrades with workspace growth (more noise). Semantic retrieval improves with workspace growth (more to correctly exclude). The gap compounds over time and is perceived by buyers as AI quality, not infrastructure — making it difficult to counter. The /ask plugin command is the fastest path to demonstrating this advantage.

## Description
Competitors injecting naive context (full snapshot or role-filtered markdown dump) get worse as workspaces mature — more data means more noise in the context window. Stratafy with semantic retrieval gets better as workspaces mature — more data means more for the similarity search to correctly exclude.

This is a compounding structural advantage, not a feature advantage. It is invisible to the buyer — they experience it as "Stratafy's AI feels smarter," not "Stratafy's context assembly is architecturally superior." That invisibility is itself strategic: it can't be countered with a feature announcement.

The /ask plugin command surfaces this advantage in its most demonstrable form — a single Claude command that answers from live workspace context with entity-level precision. This is also the lowest-cost proof of concept for the full Context Intelligence v2 architecture, requiring only search_workspace + a grounding system prompt, shippable before the April fundraise window.
