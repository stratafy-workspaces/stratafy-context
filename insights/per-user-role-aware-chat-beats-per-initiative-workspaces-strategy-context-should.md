# Per-user role-aware chat beats per-initiative workspaces — strategy context should be ambient

## Summary
Adapting Aligned's per-deal workspace model to strategy revealed a better pattern: instead of workspace links per initiative, provide each user a single role-aware chat interface where strategic context (Stratafy) and operational tasks (Linear) are unified automatically. The workspace comes to the user, not the other way around. Every task carries strategic rationale. The bidirectional loop becomes ambient.

## Description
The Aligned "one link per deal" pattern initially seemed transferable to Stratafy as "one link per initiative." But strategy isn't a discrete bounded object like a deal — initiatives interconnect with assumptions, risks, decisions, and other initiatives. A per-initiative workspace fragments exactly the context Stratafy exists to unify. The correct architecture is a single role-aware chat interface per user where: (1) the system already knows who the user is and their role, (2) strategic context from Stratafy is automatically scoped to their responsibilities, (3) operational tasks from Linear are connected to strategic rationale, (4) agents operating in their domain are visible. This means every Linear task carries strategic context — a team member can ask "why does this matter?" and get the assumption chain, risk implications, and strategic rationale without navigating anywhere. The bidirectional loop becomes ambient rather than activated. This is the MCP architecture's natural expression and a genuine differentiator: no one else is building role-aware strategic-operational context unification through a conversational interface.
