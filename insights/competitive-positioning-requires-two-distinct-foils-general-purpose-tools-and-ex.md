# Competitive positioning requires two distinct foils — general-purpose tools AND existing strategy software

## Summary
Context layer document v2 articulates the dual competitive positioning: vs general-purpose tools (can't structure strategy as queryable architecture) AND vs existing strategy software like Cascade/Quantive (Mode 1 workbenches, not infrastructure layer). The strategy software foil is the more nuanced argument investors will probe.

## Description
The context layer document v2 positions Stratafy against two separate competitive foils:

1. General-purpose tools (Notion, Docs, Confluence): Store information but don't structure it as queryable, relational, hierarchical architecture. Can't serve different resolutions to different consumers from a single source.

2. Existing strategy software (Cascade, Quantive, Perdoo, Workboard): Mode 1 products — better strategy workbenches you visit. They don't expose strategy as a consumable layer for other tools/agents/external parties. No context generation at different resolutions. No MCP. No external consumption. "Better filing cabinet, not infrastructure."

The distinction matters because investors will ask "how is this different from [strategy tool]?" and the answer is architectural, not functional: a filing cabinet stores your files, an operating system provides the environment every application runs on.

This dual-foil positioning should be consistent across all materials: pitch deck, demo narrative, methodology content. The documents/tools foil is obvious. The strategy software foil requires a more nuanced argument that should be rehearsed.
