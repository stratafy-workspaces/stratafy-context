# Semantic-first context assembly is Stratafy's structural AI quality moat — architecture validated, shippable before fundraise

## Summary
Query-aware semantic retrieval via pgvector replaces role-based context injection. Validated by live testing. Creates a compounding moat that widens as workspaces grow. Demonstrable via /ask plugin command before April fundraise. Three supporting insights logged: architecture reframe, vector DB validation, competitive compounding effect.

## Description
Three findings from a single architecture session combine into a coherent strategic position. (1) The current context system compresses by role, but should compress by query intent — role is the access ceiling, the query is the floor, and only one gate is currently applied. (2) The vector DB (pgvector via search_workspace) is already the right mechanism for query-intent routing, validated by live testing with a concrete similarity-to-depth mapping: >0.55 → full, 0.42–0.55 → executive, 0.35–0.42 → brief, <0.35 → exclude. No additional classification infrastructure needed. (3) The resulting architecture is a compounding moat — naive competitors degrade with workspace growth, semantic retrieval improves with it, and buyers perceive this as AI quality rather than infrastructure, making it impossible to counter with a feature announcement. The /ask plugin command is the minimum viable demonstration of this full architecture, shippable before the April fundraise window at near-zero cost.
