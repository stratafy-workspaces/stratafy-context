# Domain-per-plugin beats layer-per-plugin for GTM

## Summary
Domain-scoped plugins create natural purchasing units mapped to buyer roles, bundling all AI capabilities per business function.

## Description
Packaging AI tools by business domain (strategy, finance, people) rather than by technical layer (chat, MCP, automation) creates natural purchasing units. Each domain plugin is a standalone value proposition that maps to a buyer's job title, making pricing and sales conversations intuitive. This also means a single plugin can bundle chat prompts, MCP tools, and automations together rather than selling those capabilities separately.
