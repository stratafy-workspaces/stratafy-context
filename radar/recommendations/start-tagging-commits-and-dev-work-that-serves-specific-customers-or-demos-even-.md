## Description
Start tagging commits and dev work that serves specific customers or demos. Even a simple convention like `feat(stir):` or `feat(demo):` in commit messages would make it possible to measure how much dev effort is customer-facing versus platform-building. This creates visibility into the balance between building and selling — critical for a solo founder who needs to do both. The weekly dev review can then track the ratio and flag when it drifts too far in either direction.
