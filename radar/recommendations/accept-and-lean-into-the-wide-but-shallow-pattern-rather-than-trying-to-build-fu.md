# Recommendation

## Description
Accept and lean into the wide-but-shallow pattern rather than trying to build full Workbench UI for every system. The strategy thesis says Mode 2 (conversational/MCP) is where the richest co-working happens. For systems like Radar, Assumptions, Risks, and Feedback, the MCP interface may be the primary interface — the Workbench only needs viewing/auditing capability, not full CRUD UI. This saves significant dev time and validates the context layer architecture. The exception is the core Strategy and Foundation layers which need solid Mode 1 experiences for onboarding and architectural work.
