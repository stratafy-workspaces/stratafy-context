# Website Content Architecture Implementation

## Description
8-week implementation of hub-and-spoke content architecture and "Why Strategies Fail" navigation structure. Pre-pitch website launch priority.

ARCHITECTURE:
- Parent Hub: /why-strategies-fail (canonical pillar, 10+ blog spokes)
- Sub-Hub 1: /why-strategies-fail/ai-factor (6-8 spokes)
- Sub-Hub 2: /why-strategies-fail/execution-principles (5-7 spokes)
- /methodology (static page)

NAVIGATION: "Why Strategies Fail ▾" dropdown with two sub-items. Parent label clickable to hub page. "The Execution Gap" reserved for Phase 2/3.

8-WEEK TIMELINE:
Week 1: Publish parent hub, update 10+ spokes with links, homepage CTA banner
Week 2-3: Publish Methodology, draft/publish AI Factor sub-hub, update nav dropdown
Week 4-5: Publish Execution Principles sub-hub, update spokes, draft planned content
Week 6-7: Cross-hub links, activate full dropdown
Week 8: Link audit, sitemap submission, schema verification, mobile testing

KEY DECISIONS:
- "Why Strategies Fail" chosen over "The Execution Gap" for Phase 1 (provocative, high-volume, self-explanatory)
- Streamlined dropdown (parent clickable, only 2 sub-items shown)
- Natural keyword targets per sub-hub (not forced inheritance from parent slug)
- Investor journey explicitly mapped: About/Methodology → parent hub → sub-hubs → /demo
